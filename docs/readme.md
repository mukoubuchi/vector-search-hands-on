This directory contains source files for documentation generated by MkDocs.

├── feedback.md               # Participant feedback form

├── ja/                       # Japanese translations
│   ├── readme.md             # Japanese version of this file
│   ├── index.md              # Home page
│   ├── preparation.md        # Preparation
│   ├── part1.md              # Part 1: Experience Vector Search
│   ├── part2.md              # Part 2: Add features with IBM Bob
│   ├── part3.md              # Part 3: Verification and Review
│   ├── summary.md            # Summary
│   ├── feedback.md           # Participant feedback form
│   └── translation-sync.md   # Internal translation sync guide

## Stylesheet Structure

### extra.css

Main CSS file. Imports each module.

### typography.css

- Heading styles (h1, h2, h3)
- Paragraph, list, and link styles
- Keyboard key display styles
- Definition list styles

### navigation.css

- Header and tab styles
- Sidebar and TOC styles
- Back-to-top button
- Scrollbar styles

### code.css

- Code block styles
- Inline code styles
- Syntax highlight color settings
- Language-specific code styles (Bash, Python, Properties, etc.)

### components.css

- Search box styles
- Task list styles
- Admonition styles
- Tab content styles

### feedback.css

- Feedback form layout and copy panel styles

### language-switcher.css

- Language selector display adjustments

## JavaScript Structure

### site-config.js

Defines shared top-level page paths and supported languages.

### search.js

Controls search box behavior:

- Closes search when clicking outside the search box
- Filters search results to the current language and top-level pages

### navigation.js

Controls navigation functionality:

- Show/hide back-to-top button
- Behavior based on scroll position

### toc-indicator.js

Highlights the section currently on screen with a segment on the table-of-contents rail, positioned from live geometry as the page scrolls.

### tasks.js

Provides task list functionality:

- Saves checkbox state to localStorage
- Restores state on page reload

### feedback.js

Builds a Slack-ready feedback message and copies it to the clipboard.

### syntax-highlight.js

Enhances code block syntax highlighting:

- Highlights the first word of Bash commands
- Properly highlights numbers and comments in Properties files

### language-switcher.js

Normalizes language switcher links when the site is hosted under a GitHub Pages project path.

### vendor/mermaid.min.js

Vendored Mermaid build (MIT) that renders the Part 1 diagrams, bundled so the site works without internet access. The pinned version and update procedure are noted in `mkdocs.yml`.

## Benefits of Modularization

1. **Improved maintainability**: Each feature is separated into independent files, making fixes easier
2. **Improved readability**: Smaller file sizes make code easier to read
3. **Reusability**: Only required features can be selected and used
4. **Easier debugging**: Easier to identify the problematic feature

## Customization

### Changing Styles

To change specific styles, edit the corresponding CSS module:

- Typography → `typography.css`
- Navigation → `navigation.css`
- Code blocks → `code.css`
- UI components → `components.css`

### Adding Features

To add new JavaScript functionality:

1. Create a new file in the `javascripts/` directory
2. Add it to the `extra_javascript` section in `mkdocs.yml`

Example:

```yaml
extra_javascript:
  - javascripts/site-config.js
  - javascripts/search.js
  - javascripts/navigation.js
  - javascripts/tasks.js
  - javascripts/feedback.js
  - javascripts/syntax-highlight.js
  - javascripts/language-switcher.js
  - javascripts/your-new-feature.js  # new file
```

When adding or removing top-level pages or languages, update `javascripts/site-config.js`. The search filter and language switcher both read this shared site configuration.

## Development Guidelines

### CSS Guidelines

- Use CSS variables (`var(--md-*)`) to maintain consistency with Material Theme
- Annotate the purpose of each section in comments
- Consider responsive design (use media queries)

### JavaScript Guidelines

- Describe functionality at the top of each file with JSDoc comments
- Use `DOMContentLoaded` event for initialization
- Use `setTimeout` as needed to wait for Material Theme initialization

## Build and Preview

### Local Preview

```bash
# Run from project root
mkdocs serve
```

Access at `http://localhost:8000` in your browser.

### Production Build

```bash
# Run from project root
mkdocs build
```

Built files are output to the `site/` directory.

## Troubleshooting

### Styles not applied

1. Clear browser cache
2. Restart `mkdocs serve`
3. Check CSS file import order

### JavaScript not working

1. Check browser console for errors
2. Check `extra_javascript` section in `mkdocs.yml`
3. Verify file paths are correct
