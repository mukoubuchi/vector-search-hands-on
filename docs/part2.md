v# Part 2: IBM Bob で機能を追加しよう

このパートでは、IBM Bob を使って、Vector Search アプリケーションに新しい機能を追加します。

## このパートのゴール

- IBM Bob の使い方を学ぶ
- 自然言語で指示を出してコードを生成する
- 3 つの新機能を追加する

## IBM Bob とは？（復習）

**IBM Bob** = AI がコーディングをサポートしてくれる開発ツール

**できること**:

- 自然言語で「こういう機能が欲しい」と伝える
- IBM Bob が自動的にコードを書いてくれる
- コードの説明もしてくれる

**メリット**:

- コーディングの手間を削減
- 開発時間が大幅に短縮
- 高品質なコードが生成される

## 追加する機能

このパートでは、以下の 3 つの機能を追加します:

1. **商品画像の表示**
2. **価格フィルター**
3. **レコメンド理由の表示**

## 機能 1: 商品画像の表示

!!! info "なぜこの機能が必要？"
    現在の検索結果は文字だけです。商品画像があると視覚的に分かりやすく、ユーザー体験が向上します。

### ステップ 1: IBM Bob を開く

IBM Bob の画面下部のチャット入力欄をクリック

### ステップ 2: Code モードに切り替え

1. 画面右下の「Mode」セレクターをクリック
2. 「Code」を選択

**Code モード** = コードを書くための専用モード

??? note "他のモードでも実行可能"
    このタスクは以下のモードでも実行できます:
    
    - **Advanced モード**: コード編集 + 追加ツール（MCP、Browser）
    - **Vector Search Builder モード**: Vector Search 特化（このハンズオンで使用中）
    
    Code モードを選択することで、モード切り替えの基本操作を学べます。

### ステップ 3: IBM Bob に指示を出す

チャット入力欄に以下を入力して Enter:

```
検索結果に商品画像を表示してください
```

**ポイント**:

- 具体的に何をしたいか伝える

### ステップ 4: IBM Bob の応答を待つ

IBM Bob が自動的に以下を行います:

1. 指示を理解
2. 関連ファイルを探す
3. コードを生成
4. 説明を表示

### ステップ 5: IBM Bob の提案を確認

IBM Bob が以下のような提案をします:

```
検索結果に商品画像を表示する機能を追加します。
```

変更内容:

- **`app.py`**: レスポンスに **`image_url`** フィールドを追加
- 商品データから画像 URL を取得
- 検索結果に画像 URL を含める

### ステップ 6: 変更を承認

1. IBM Bob の提案を読む
2. 「承認」ボタンをクリック
3. ファイルに変更が適用される

### ステップ 7: 動作確認

1. Swagger UI を開く（**`http://localhost:8000/docs`**）
2. アプリケーションを再起動
    1. ターミナルで ++ctrl+c++ （停止）
    2. IBM Bob に「デモアプリケーションを起動して」と依頼、または **`python app.py`** を実行（[:material-play-circle: 起動方法](../part1/#app-restart)）
3. 検索を実行:

   ```json
   {
     "query": "赤いスニーカー"
   }
   ```

4. 結果を確認:

   ```json
   {
     "results": [
       {
         "product_name": "赤いランニングシューズ",
         "image_url": "https://example.com/images/red-shoes.jpg",
         "similarity_score": 0.92,
         "price": 8900
       }
     ]
   }
   ```

**確認ポイント**: **`image_url`** フィールドが追加されている

### 機能 1 完了チェック

- [ ] IBM Bob に指示を出せた
- [ ] IBM Bob がコードを生成した
- [ ] 変更を承認した
- [ ] 検索結果に **`image_url`** が表示される

## 機能 2: 価格フィルター

### なぜこの機能が必要？

価格帯で絞り込めると、予算に合った商品を探せます。

### ステップ 1: IBM Bob に指示を出す

チャット入力欄に以下を入力して Enter:

```text
価格帯でフィルターできる機能を追加してください。
最小価格と最大価格を指定できるようにしてください。
```

### ステップ 2: IBM Bob の提案を確認

IBM Bob が以下のような提案をします:

```
価格フィルター機能を追加します。
```

変更内容:

- **`app.py`**: **`min_price`** と **`max_price`** パラメータを追加
- 検索結果を価格帯でフィルタリング

### ステップ 3: 変更を承認

「承認」ボタンをクリック

### ステップ 4: 動作確認

1. アプリケーションを再起動
    1. ターミナルで ++ctrl+c++ （停止）
    2. IBM Bob に「デモアプリケーションを起動して」と依頼、または **`python app.py`** を実行（[:material-play-circle: 起動方法](../part1/#app-restart)）
2. 検索を実行:

   ```json
   {
     "query": "スニーカー",
     "min_price": 5000,
     "max_price": 10000
   }
   ```

3. 結果を確認: 5000 円〜10000 円の商品のみ表示される

### 機能 2 完了チェック

- [ ] 変更を承認した

レコメンド理由表示機能を追加します。

## FAQ

??? question "Q1: IBM Bob が応答しない"

    対処法:
    
    1. インターネット接続を確認
    2. IBM Bob を再起動

??? question "Q2: 変更が反映されない"

    対処法:
    
    1. ファイルが保存されているか確認
    2. アプリケーションを再起動
        1. ターミナルで ++ctrl+c++ （停止）
        2. IBM Bob に「デモアプリケーションを起動して」と依頼、または **`python app.py`** を実行（[:material-play-circle: 起動方法](../part1/#app-restart)）
    3. ブラウザをリロード

??? question "Q3: エラーが表示される"

    対処法:
    
    1. エラーメッセージをコピー
    2. IBM Bob に **「このエラーを解決して」** と依頼
