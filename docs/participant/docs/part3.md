## 1. アプリケーションの動作確認（10 分）

### 📝 事前確認

動作確認を始める前に、以下を確認してください：

1. **アプリケーションが起動していること**

   - ターミナルで `python app.py` が実行中であること
   - エラーメッセージが表示されていないこと

2. **Swagger UI にアクセスできること**

   - ブラウザで `http://localhost:8000/docs` を開く
   - API ドキュメントが表示されること

### 🔍 Swagger UI の使い方

Swagger UI は、API を視覚的にテストできるツールです。以下の手順で操作します：

#### 基本操作

1. **エンドポイントを選択**

   - `/search` エンドポイントをクリック
   - エンドポイントの詳細が展開される

2. **「Try it out」ボタンをクリック**

   - 入力フィールドが編集可能になる

3. **パラメータを入力**

   - `query` フィールドに検索クエリを入力
   - 必要に応じて `filters` を設定

4. **「Execute」ボタンをクリック**

   - API が実行される
   - レスポンスが下部に表示される

5. **結果を確認**

   - `Response body` セクションで JSON レスポンスを確認
   - `Response headers` でヘッダー情報を確認

### 🧪 テストケース 1: 基本的な商品検索

#### 目的

Part 2 で実装した「商品画像と類似度スコアの表示」機能を確認します。

#### 手順

1. **Swagger UI で `/search` エンドポイントを開く**

2. **「Try it out」をクリック**

3. **以下のパラメータを入力**

   ```json
   {
     "query": "赤いスニーカー"
   }
   ```

4. **「Execute」をクリック**

#### 確認ポイント

レスポンスの JSON を確認し、以下が含まれていることを確認します：

```json
{
  "results": [
    {
      "product_name": "商品名",
      "image_url": "https://...",           // ✅ 画像 URL が表示される
      "similarity_score": 0.92,             // ✅ 類似度スコアが表示される
      "price": 8900,
      "category": "スニーカー"
    }
  ]
}
```

**期待される結果**:

- ✅ `image_url` フィールドに画像 URL が含まれている
- ✅ `similarity_score` が 0.0〜1.0 の範囲で表示される
- ✅ スコアが高い順（類似度が高い順）に並んでいる

#### トラブルシューティング

**画像が表示されない場合**:

- `image_url` フィールドが存在するか確認
- URL が正しい形式か確認（`https://` で始まる）
- サンプルデータに画像 URL が含まれているか確認

**類似度スコアが表示されない場合**:

- `similarity_score` フィールドが存在するか確認
- Part 2 の実装が正しく反映されているか確認
- アプリケーションを再起動してみる

**フィルターが機能しない場合**:

- フィルター条件が厳しすぎる可能性
- サンプルデータに該当する商品があるか確認
- 価格帯やカテゴリ名を調整してみる

## 2. IBM Bob によるコードレビュー（5 分）

### 🎯 コードレビューの目的

IBM Bob の `/review` コマンドを使用して、以下を確認します：

- コードの品質（可読性、保守性）
- パフォーマンスの問題

- ベストプラクティスへの準拠

### 📝 `/review` コマンドの実行手順

#### ステップ 1: IBM Bob を開く

1. **IBM Bob IDE のチャット画面を開く**

   - 画面下部のチャット入力欄をクリック

2. **現在のモードを確認**

   - 右下の「Mode」セレクターを確認
   - 「Code」モードになっていることを確認

#### ステップ 2: レビュー対象ファイルを開く

1. **レビューしたいファイルを開く**

   - 例: `app.py`、`search.py` など
   - Part 2 で編集したファイルを開く

2. **ファイルが IBM Bob に認識されていることを確認**

   - ファイルタブが表示されている
   - コードが表示されている

#### ステップ 3: `/review` コマンドを実行

1. **チャット入力欄に以下を入力**

   ```text
   /review
   ```

2. **Enter キーを押す**

3. **IBM Bob がコードを分析するまで待つ**

   - 通常 10〜30 秒程度
   - 「Analyzing code...」のようなメッセージが表示される

### 📋 レビュー結果の確認

IBM Bob は以下のような形式でレビュー結果を表示します：

#### レビュー結果の例

```text

【良い点】

✓ エラーハンドリングが適切に実装されている
✓ 型ヒント（Type Hints）が使用されている
✓ ドキュメント文字列（Docstrings）が充実している

【改善提案】

  - ベクトル検索結果のキャッシュ機構の追加を検討

⚠️ セキュリティ
  - API キーの環境変数化を推奨

⚠️ 保守性
  - ログ出力の強化（デバッグ情報の追加）
  - テストケースの追加を推奨
```

### 🔍 レビュー結果の読み方

#### 1. 良い点（✓）

IBM Bob が評価したコードの良い点です。以下のような観点で評価されます：

- **設計**: API 設計、アーキテクチャ
- **実装**: コードの書き方、パターンの使用
- **ドキュメント**: コメント、説明の充実度

**例**:

- ✓ RESTful な API 設計 → 業界標準に準拠している
- ✓ 型ヒントの使用 → コードの可読性が高い
- ✓ エラーハンドリング → 堅牢な実装

#### 2. 改善提案（⚠️）

IBM Bob が提案する改善点です。以下のカテゴリに分類されます：

##### パフォーマンス

**意味**: 処理速度やリソース使用量の最適化

**例**:

- キャッシュ機構の追加
- データベースクエリの最適化
- 不要な処理の削減

**対応方法**:

```python
# 改善前
results = search_vector_db(query)  # 毎回検索

# 改善後（キャッシュ使用）
if query in cache:
    results = cache[query]
else:
    results = search_vector_db(query)
    cache[query] = results
```

##### セキュリティ

**意味**: セキュリティリスクの軽減

**例**:

- API キーのハードコード回避
- 入力値のバリデーション
- SQL インジェクション対策

**対応方法**:

```python
# 改善前
API_KEY = "sk-1234567890"  # ハードコード

# 改善後（環境変数使用）
import os
API_KEY = os.getenv("WATSONX_API_KEY")
```

##### 保守性

**意味**: コードの読みやすさ、メンテナンスのしやすさ

**例**:

- ログ出力の追加
- テストケースの作成
- コメントの充実

**対応方法**:

```python
# 改善前
def search(query):
    return results

# 改善後（ログ追加）
import logging

def search(query):
    logging.info(f"検索クエリ: {query}")
    results = perform_search(query)
    logging.info(f"検索結果: {len(results)}件")
    return results
```

### 💡 改善提案への対応

#### 優先順位の付け方

1. **🔴 高優先度**: セキュリティリスク、重大なバグ

## 3. よくある質問とトラブルシューティング

### 🤔 よくある質問（FAQ）

#### Q1: Swagger UI にアクセスできない

**症状**: `http://localhost:8000/docs` にアクセスしても表示されない

**原因と対処法**:

1. **アプリケーションが起動していない**

   ```bash
   # ターミナルで確認
   python app.py
   ```

   - 「Running on <http://localhost:8000>」と表示されるか確認

2. **ポートが使用中**

   ```bash
   # 別のポート（例: 8001）で起動
   python app.py --port 8001
   ```

   - `http://localhost:8001/docs` でアクセス

3. **ファイアウォールの問題**

   - セキュリティソフトがポート 8000 をブロックしている可能性
   - 一時的に無効化するか、ポートを許可

#### Q2: 検索結果が 0 件になる

**症状**: どんなクエリでも結果が返ってこない

**原因と対処法**:

1. **サンプルデータが投入されていない**

   ```bash
   # データ投入スクリプトを実行
   python scripts/load_sample_data.py
   ```

2. **Milvus への接続エラー**

   - ターミナルのエラーメッセージを確認
   - TechZone 環境の接続情報を再確認

3. **埋め込みモデルの問題**

   - watsonx.ai の API キーが正しいか確認
   - API の利用制限に達していないか確認

#### Q3: 類似度スコアが表示されない

**症状**: 検索結果は返るが、`similarity_score` が含まれていない

**原因と対処法**:

1. **Part 2 の実装が反映されていない**

   - アプリケーションを再起動
   - コードの変更が保存されているか確認

2. **レスポンスの構造を確認**

   ```python
   # app.py で以下を確認
   return {
       "results": [
           {
               "similarity_score": score,  # この行があるか
               ...
           }
       ]
   }
   ```

#### Q4: フィルターが機能しない

**症状**: フィルター条件を指定しても、すべての商品が表示される

**原因と対処法**:

1. **JSON の構文エラー**

   ```json
   // ❌ 間違い（カンマが多い）
   {
     "query": "スニーカー",
     "filters": {
       "price_range": {
         "min": 5000,
         "max": 10000,
       }
     }
   }

   // ✅ 正しい
   {
     "query": "スニーカー",
     "filters": {
       "price_range": {
         "min": 5000,
         "max": 10000
       }
     }
   }
   ```

2. **フィルター実装の確認**

   - Part 2 でフィルター機能を実装したか確認
   - IBM Bob に「フィルターが機能しない」と相談

#### Q5: `/review` コマンドが動作しない

**症状**: `/review` を実行してもレビュー結果が表示されない

**原因と対処法**:

1. **ファイルが開かれていない**

   - レビュー対象のファイルを IBM Bob で開く
   - ファイルタブが表示されているか確認

2. **モードが間違っている**

   - 右下の「Mode」セレクターを確認
   - 「Code」モードに切り替える

3. **IBM Bob の再起動**

   - IBM Bob IDE を再起動
   - プロジェクトフォルダを再度開く

### 🔧 トラブルシューティングの基本手順

問題が発生した場合、以下の順序で確認します：

#### ステップ 1: エラーメッセージを確認

1. **ターミナルのエラーメッセージを読む**

   - エラーの種類を特定
   - エラーが発生した行番号を確認

2. **ブラウザの開発者ツールを確認**

   - F12 キーで開発者ツールを開く
   - Console タブでエラーを確認

#### ステップ 2: 基本的な確認

- [ ] アプリケーションが起動しているか
- [ ] 正しい URL にアクセスしているか
- [ ] インターネット接続は正常か
- [ ] TechZone 環境にアクセスできるか

#### ステップ 3: IBM Bob に相談

問題が解決しない場合、IBM Bob に相談します：

```text
「検索結果が 0 件になります。エラーメッセージは以下の通りです：
[エラーメッセージをコピー]」
```

IBM Bob が原因を分析し、解決策を提案してくれます。
