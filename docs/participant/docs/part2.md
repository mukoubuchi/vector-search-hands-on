# Part 2: IBM Bobで機能を追加しよう

**所要時間**: 20分

このパートでは、IBM Bobを使って、Vector Searchアプリケーションに新しい機能を追加します。

## このパートのゴール

- IBM Bobの使い方を学ぶ
- 日本語で指示を出してコードを生成する
- 3つの新機能を追加する

## IBM Bobとは？（復習）

**IBM Bob** = AIがコーディングをサポートしてくれる開発ツール

**できること**:

- 日本語で「こういう機能が欲しい」と伝える
- IBM Bobが自動的にコードを書いてくれる
- コードの説明もしてくれる

**メリット**:

- プログラミング経験不要
- 開発時間が大幅に短縮
- 高品質なコードが生成される

## 追加する機能

このパートでは、以下の3つの機能を追加します：

1. **商品画像の表示**（7分）
   - 検索結果に商品画像を表示

2. **価格フィルター**（7分）
   - 価格帯で絞り込み

3. **レコメンド理由の表示**（6分）
   - なぜその商品がおすすめなのかを表示

## 機能1: 商品画像の表示（7分）

!!! info "説明: なぜこの機能が必要？"
    現在の検索結果は文字だけです。商品画像があると：
    
    - 視覚的に分かりやすい
    - ユーザー体験が向上
    - 商品の特徴が一目で分かる

!!! example "実践: 商品画像機能を追加しよう"

### ステップ1: IBM Bobを開く

1. **IBM Bob IDEの画面下部を確認**
   - チャット入力欄があります

2. **チャット入力欄をクリック**
   - カーソルが点滅します

### ステップ2: Codeモードに切り替え

1. **画面右下の「Mode」セレクターを確認**
   - 現在のモードが表示されています

2. **「Mode」をクリック**
   - モード選択メニューが表示されます

3. **「Code」を選択**
   - Codeモードに切り替わります

**Codeモードとは？**:
- コードを書くための専用モード
- コード生成、編集、レビューができる

### ステップ3: IBM Bobに指示を出す

チャット入力欄に以下を入力してEnterキーを押します：

```
検索結果に商品画像を表示してください
```

**ポイント**:

- 日本語で自然に書く
- 「〜してください」と丁寧に依頼
- 具体的に何をしたいか伝える

### ステップ4: IBM Bobの応答を待つ

IBM Bobが以下のことを自動的に行います：

1. **指示を理解**
   - 「商品画像を表示」という要求を理解

2. **関連ファイルを探す**
   - どのファイルを修正すべきか判断

3. **コードを生成**
   - 必要なコードを書く

4. **説明を表示**
   - 何をしたかを説明

**所要時間**: 10〜30秒程度

### ステップ5: IBM Bobの提案を確認

IBM Bobが以下のような提案をします：

```
検索結果に商品画像を表示する機能を追加しました。

変更内容:
- app.py: レスポンスにimage_urlフィールドを追加
- 商品データから画像URLを取得
- 検索結果に画像URLを含める

変更したファイル:
- app.py (15行追加)
```

**確認ポイント**:
- どのファイルが変更されたか
- 何行追加されたか
- 変更内容の説明

### ステップ6: 変更を承認

1. **IBM Bobの提案を読む**
   - 変更内容を理解する

2. **「承認」または「Apply」ボタンをクリック**
   - 変更が適用されます

3. **ファイルが自動的に保存される**
   - 変更が反映されます

### ステップ7: 動作確認

1. **Swagger UIを開く**
   - ブラウザで `http://localhost:8000/docs`

2. **アプリケーションを再起動**
   - ターミナルで `Ctrl + C`（停止）
   - `python app.py`（再起動）

3. **検索を実行**
   ```json
   {
     "query": "赤いスニーカー"
   }
   ```

4. **結果を確認**
   ```json
   {
     "results": [
       {
         "product_name": "赤いランニングシューズ",
         "image_url": "https://example.com/images/red-shoes.jpg",
         "similarity_score": 0.92,
         "price": 8900
       }
     ]
   }
   ```

**確認ポイント**:
- `image_url`フィールドが追加されている
- URLが正しい形式（`https://`で始まる）

- [ ] IBM Bobがコードを生成した
- [ ] 変更を承認した

## 機能2: 価格フィルター（7分）

### なぜこの機能が必要？

価格帯で絞り込めると：
- 予算に合った商品を探せる
- 検索結果が絞り込まれる
- ユーザーの満足度が向上

### ステップ1: IBM Bobに指示を出す

チャット入力欄に以下を入力してEnterキーを押します：

```
価格帯でフィルターできる機能を追加してください。
最小価格と最大価格を指定できるようにしてください。
```

**ポイント**:
- 具体的に何をしたいか説明
- 「最小価格と最大価格」と詳細を指定
- 複数の文で説明してもOK

### ステップ2: IBM Bobの提案を確認

IBM Bobが以下のような提案をします：

```
価格帯フィルター機能を追加しました。

変更内容:
- app.py: filtersパラメータを追加
- price_rangeフィールドでmin/maxを指定可能
- Vector Searchとフィルターを組み合わせ

使用例:
{
  "query": "スニーカー",
  "filters": {
    "price_range": {
      "min": 5000,
      "max": 10000
    }
  }
}
```

### ステップ3: 変更を承認

1. **提案内容を確認**
2. **「承認」ボタンをクリック**
3. **ファイルが保存される**

### ステップ4: 動作確認

1. **アプリケーションを再起動**
   - ターミナルで `Ctrl + C`
   - `python app.py`

2. **Swagger UIで検索**
   ```json
   {
     "query": "スニーカー",
     "filters": {
       "price_range": {
         "min": 5000,
         "max": 10000
       }
     }
   }
   ```

3. **結果を確認**
   - すべての商品の価格が5000〜10000円の範囲内
   - 範囲外の商品は表示されない

## 機能3: レコメンド理由の表示（6分）

### なぜこの機能が必要？

レコメンド理由があると：
- なぜその商品がおすすめなのか分かる
- ユーザーが納得して購入できる
- 信頼性が向上

### ステップ1: IBM Bobに指示を出す

チャット入力欄に以下を入力してEnterキーを押します：

```
検索結果に、なぜその商品がレコメンドされたのか、
理由を表示してください。
検索クエリと商品の類似点を説明してください。
```

**ポイント**:
- 「なぜ」「理由」と明確に伝える
- 「類似点を説明」と具体的に指定
- 改行を使って読みやすく

### ステップ2: IBM Bobの提案を確認

IBM Bobが以下のような提案をします：

```
レコメンド理由の表示機能を追加しました。

変更内容:
- app.py: reasonフィールドを追加
- similar_featuresフィールドで類似点をリスト表示
- 検索クエリと商品の特徴を比較

出力例:
{
  "reason": "検索クエリ「初心者向けのカメラ」と以下の点で類似しています：",
  "similar_features": [
    "対象: 初心者・入門者向け",
    "カテゴリ: デジタルカメラ",
    "特徴: 簡単操作"
  ]
}
```

### ステップ3: 変更を承認

1. **提案内容を確認**
2. **「承認」ボタンをクリック**
3. **ファイルが保存される**

## IBM Bob活用のコツ

### 1. 具体的に指示する

**:fontawesome-solid-xmark: 悪い例**:

**:fontawesome-solid-xmark: 悪い例**:

```
レコメンド理由も表示して、デザインも変更して...

```

```

```


- 「他の方法はありますか？」

## :fontawesome-solid-circle-question: 困ったときは

### よくある質問

#### Q1: IBM Bobが応答しない

**対処法**:
1. インターネット接続を確認
2. IBM Bobを再起動
3. 講師に相談

#### Q2: 提案内容が理解できない

**対処法**:
1. IBM Bobに「この変更を詳しく説明してください」と質問
2. 講師に説明してもらう
3. 一旦スキップして次に進む

#### Q3: 変更を承認したが動作しない

**対処法**:
1. アプリケーションを再起動
2. ブラウザをリロード
3. エラーメッセージを確認
4. 講師に相談

#### Q4: 間違った変更を承認してしまった

**対処法**:
1. IBM Bobに「前の変更を取り消してください」と依頼
2. または、講師に相談
3. 元のファイルに戻す
