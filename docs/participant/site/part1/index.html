    Part 1: Vector Searchを体験しよう

    所要時間: 15分

    このパートでは、Vector Search（ベクトル検索）がどのように動作するかを実際に体験します。

    🎯 このパートのゴール

    • Vector Searchとは何かを理解する
    • 実際にVector Searchを動かしてみる
    • 「意味で検索」の便利さを体感する

    ステップ1: Vector Searchとは？（5分）

    1-1. 従来の検索の問題点

    まず、普段使っている検索の問題点を理解しましょう。

    例: ECサイトで商品を探す場合

    あなたの検索: 「赤いスニーカー」

    従来の検索結果: - ✅ 「赤いスニーカー」→ 見つかる - ❌ 「赤色のランニングシューズ」→ 見つからない - ❌ 「レッドのスポーツシューズ」→ 見つからない

    なぜ見つからない？ - 従来の検索は「文字」を探すだけ - 「赤い」と「赤色」は違う文字として扱われる - 「スニーカー」と「ランニングシューズ」も違う文字

    1-2. Vector Searchの仕組み

    Vector Searchは「意味」を理解して検索します。

    同じ例でVector Searchを使うと

    あなたの検索: 「赤いスニーカー」

    Vector Searchの結果: - ✅ 「赤いスニーカー」→ 見つかる - ✅ 「赤色のランニングシューズ」→ 見つかる（意味が似ている） - ✅ 「レッドのスポーツシューズ」→ 見つかる（意味が似ている）

    なぜ見つかる？ - Vector Searchは「意味」を理解する - 「赤い」「赤色」「レッド」→ 同じ意味と理解 - 「スニーカー」「ランニングシューズ」「スポーツシューズ」→ 似た意味と理解

    1-3. Vector Searchの仕組み（簡単に）

    Vector Searchは以下の手順で動作します：

    ステップ1: テキストを数値に変換
「赤いスニーカー」→ [0.2, 0.8, 0.1, 0.5, ...] （ベクトル）

ステップ2: 似た数値を探す
データベースから似た数値のパターンを検索

ステップ3: 結果を返す
似た意味の商品を返す

    ポイント: - 「ベクトル」= 数値の配列（例: [0.2, 0.8, 0.1, ...]） - 意味が似ていると、数値のパターンも似る - コンピュータは数値の類似度を高速に計算できる

    1-4. 実際の活用例

    Vector Searchは様々な場面で使われています：

    ECサイト

    • 「この商品に似た商品を探す」
    • 「初心者向けのカメラ」→ 入門用カメラを提案

    社内検索

    • 「この資料に似た文書を探す」
    • 「プロジェクト計画書」→ 似た計画書を検索

    カスタマーサポート

    • 「似た質問を探す」
    • 過去の回答を素早く見つける

    ステップ2: 接続テストを実行（5分）

    実際にVector Searchを動かす前に、必要なサービスに接続できるか確認します。

    2-1. ターミナルを開く

    1. IBM Bob IDEの画面下部を確認

    2. 「TERMINAL」というタブがあります

    3. TERMINALタブをクリック

    4. 黒い画面（ターミナル）が表示されます

    5. ターミナルとは？

    6. コンピュータに命令を出す画面
    7. 文字で命令を入力します

    2-2. setupフォルダに移動

    ターミナルに以下を入力してEnterキーを押します：

    cd setup

    説明: - cd = "Change Directory"（フォルダを移動する命令） - setup = 移動先のフォルダ名

    確認: - ターミナルの表示が変わります - ~/vector-search-handson/setup のような表示になればOK

    2-3. 接続テストを実行

    ターミナルに以下を入力してEnterキーを押します：

    python test_connection_simple.py

    説明: - python = Pythonプログラムを実行する命令 - test_connection_simple.py = 接続テストのプログラム

    2-4. 結果を確認

    接続テストが実行されます。以下のような結果が表示されます：

    ステップ3: Vector Searchを体験（5分）

    接続テストが成功したら、実際にVector Searchを体験しましょう。

    3-1. デモアプリケーションを起動

    方法1: IBM Bobに依頼（推奨）

    1. IBM Bobのチャット画面を開く

    2. 画面下部のチャット入力欄をクリック

    3. 以下を入力してEnter 

      setupフォルダでデモアプリケーションを起動して

    4. IBM Bobが自動的に起動

    5. ターミナルでコマンドが実行されます
    6. アプリケーションが起動します

    方法2: 手動で起動

    ターミナルに以下を入力してEnter：

    python app.py

    3-2. 起動を確認

    ターミナルに以下のような表示が出ればOK：

    INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://localhost:8000

    重要: - この画面は閉じないでください - アプリケーションが動いている間、ずっと表示されます

    3-3. Swagger UIを開く

    1. Webブラウザを開く

    2. Chrome、Firefox、Safari、Edgeなど

    3. 以下のURLにアクセス 

      http://localhost:8000/docs

    4. Swagger UIが表示される

    5. APIのドキュメントが表示されます
    6. 「Swagger UI」= APIをテストできる画面

    3-4. Swagger UIとは？

    Swagger UI は、APIを視覚的にテストできるツールです。

    できること: - APIの一覧を見る - APIを実際に実行する - 結果をすぐに確認する

    メリット: - コードを書かずにテストできる - 結果がすぐに分かる - 初心者でも使いやすい

    3-5. 検索を試してみる

    ステップ1: /searchエンドポイントを開く

    1. Swagger UI画面で「/search」を探す

    2. 緑色の「POST」ボタンの横に「/search」と書いてあります

    3. 「/search」をクリック

    4. エンドポイントの詳細が展開されます

    ステップ2: 「Try it out」をクリック

    1. 右上の「Try it out」ボタンをクリック
    2. 入力フィールドが編集可能になります

    ステップ3: 検索クエリを入力

    1. 「Request body」の欄に以下を入力 

      {
  "query": "赤いスニーカー"
}

    2. 入力内容の説明

    3. query: 検索したい言葉
    4. "赤いスニーカー": 探したい商品

    ステップ4: 「Execute」をクリック

    1. 青い「Execute」ボタンをクリック

    2. 検索が実行されます

    3. 少し待つ

    4. 数秒で結果が表示されます

    ステップ5: 結果を確認

    画面を下にスクロールすると、以下のような結果が表示されます：

    {
  "results": [
    {
      "product_name": "赤いランニングシューズ",
      "similarity_score": 0.92,
      "price": 8900,
      "category": "スニーカー",
      "description": "軽量で通気性の良いランニングシューズ"
    },
    {
      "product_name": "レッドスポーツシューズ",
      "similarity_score": 0.88,
      "price": 7500,
      "category": "スニーカー",
      "description": "カジュアルにも使えるスポーツシューズ"
    }
  ]
}

    結果の見方: - product_name: 商品名 - similarity_score: 類似度（0.0〜1.0、高いほど似ている） - price: 価格 - category: カテゴリ - description: 説明

    3-6. 色々な検索を試してみる

    以下のような検索を試してみましょう：

    例1: 初心者向けの商品を探す

    {
  "query": "初心者向けのカメラ"
}

    期待される結果: - 「入門用デジタルカメラ」 - 「ビギナー向けカメラ」 - 「簡単操作のカメラ」

    例2: ビジネス向けの商品を探す

    {
  "query": "ビジネス向けのノートパソコン"
}

    期待される結果: - 「ビジネスノートPC」 - 「オフィス用ノートパソコン」 - 「仕事用ノートPC」

    例3: 高性能な商品を探す

    {
  "query": "高性能なゲーミングPC"
}

    期待される結果: - 「ハイスペックゲーミングPC」 - 「高速ゲーミングコンピュータ」 - 「パワフルなゲーム用PC」

    3-7. Vector Searchの凄さを実感

    色々な検索を試すと、以下のことに気づくはずです：

    気づき1: 言い方が違っても見つかる - 「初心者向け」→「入門用」「ビギナー向け」も見つかる - 「高性能」→「ハイスペック」「高速」も見つかる

    気づき2: 類似度スコアが便利 - スコアが高い = より似ている - スコアが低い = あまり似ていない - 結果の信頼度が分かる

    気づき3: 説明文も考慮される - 商品名だけでなく、説明文の意味も理解 - より正確な検索結果
